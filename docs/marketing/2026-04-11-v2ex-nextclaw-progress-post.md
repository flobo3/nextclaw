# [开源] NextClaw 更新：微信、远程访问、Codex、Claude Code 接进来之后，我开始补“真能用”的部分

项目地址：

- GitHub: https://github.com/Peiiii/nextclaw
- 官网: https://nextclaw.io/
- 文档: https://docs.nextclaw.io/
- 愿景: https://docs.nextclaw.io/zh/guide/vision

大家好，我是 NextClaw 的作者。

`2026-03-24` 我在 V2EX 发过一版 NextClaw 的功能更新帖，当时主要讲的是：

- 微信 clawbot
- 远程访问 / 授权分享
- Codex
- Claude Code
- 开启认证

到现在这两周多，仓库里大约又增加了 `357` 个 commit。

所以这次不是把上次讲过的功能再换个标题发一遍，而是想讲清楚一件事：

这些能力接进来之后，我这段时间主要在补“真能用”的部分。

如果上次你对 NextClaw 的印象是：

- 能接微信
- 能远程访问
- 能把 Codex / Claude Code 接进来

那这次更想更新的一句话是：

它开始不只是“能接进来”，而是继续往“能长期用、少折腾、能放进真实工作流”这个方向走了。

顺便补一句，如果你还没看过前情，NextClaw 的定位一直很简单：

- 一条命令启动
- 用 UI 配置模型和渠道
- 同一个入口里接聊天、渠道、自动化、Agent 能力
- 尽量别让用户自己去拼一堆碎插件和命令

这两周多，我觉得最值得拿出来讲的，主要是下面几件事。

## 1. 远程访问从“能用”继续走向“产品化”

3 月 24 日那版里，我已经讲过 remote access 和授权分享，但那时候更偏“这个能力有了”。

这之后继续补的是产品层的闭环：

- 更正式的 `Remote Access` 页面
- 账号入口
- 设备状态
- 诊断
- 服务控制
- `NextClaw Web` 打开链路

对我来说，这个变化的重点不是多了一个页面，而是本地实例开始更像“这台设备上的 AI 工作台”，而不是只能在当前机器浏览器里开着。

## 2. Codex 和 Claude Code 这条线继续补深了

上次讲的是“可以接进来”，这次补的则是“接进来以后别在关键地方掉链子”。

`Codex` 这边，这两周继续补了：

- 模型路由
- 图片输入
- 本地插件开发验证链路

`Claude Code` 这边，继续补了：

- marketplace 安装链路
- provider 路由
- 和 OpenAI-compatible provider 之间的 bridge

也就是说，如果你之前对这两条能力的判断是“概念挺好，但可能只是先挂进去”，那现在至少已经开始往“真实工作流里能继续用下去”这个方向走了。

## 3. 多 Agent / child session 开始更成形了

这段时间我们把 child session / subagent 这条链路往前推了不少，也做了一轮更明确的对外版本升级。

我更在意的不是“能不能喊 multi-agent”，而是它是不是开始具备：

- 独立会话
- 上下文
- 状态归属
- 交互链路

现在这部分相比 3 月 24 日那一版已经更像一个会继续长下去的产品方向，而不只是挂在文案里的概念。

## 4. 桌面分发和安装路线又往前补了一大截

如果一个产品想做“默认入口”，就不能一直停留在只适合命令行用户的阶段。

所以这段时间我们一直在补桌面路线。

Linux 这边已经继续补到了：

- `.deb`
- APT 仓库

桌面端也开始推进 launcher / bundle 更新基础层。

我的想法很简单：不是所有人都愿意每次都手动重装、手动拼环境，这些基础台阶迟早得补。

## 5. 日常使用链路继续从“能演示”往“能常用”收

聊天这一层，这段时间也不是没动。

这两周继续收的主要是这些：

- 消息流
- 工具卡片
- 图片与附件
- 文件预览和 diff
- 会话列表
- 项目上下文
- 模型选择
- 输入框体验

我不太想把这类更新写成花哨功能，因为对我来说它们更像产品基建。很多时候，真正让人留下来的不是一个新按钮，而是整个过程开始更稳、更顺、更可预期。

## 6. 渠道、附件、自动化和能力层还在继续长

这条线也一直没停。

微信、飞书相关链路继续在补图片和附件；Cron、Heartbeat 继续往统一 runtime 收敛；Marketplace、MCP、skill 这条能力层也还在继续长。

所以如果你在 3 月 24 日那一版之后就没继续关注，这次我更想补一句：

它开始不只是一个“能演示很多能力的 AI 项目”，而是慢慢更像一个统一入口了。

还没有到位，但方向已经比 3 月 24 日那会儿更明确了。

## 现在仍然比较克制的地方

我也不想把话说满。

目前有些能力还在继续产品化，不是每一块都已经打磨到最简。

比如：

- 桌面更新链路现在更多还是基础层先补齐
- `Codex / Claude Code` 还会继续顺使用细节
- 整体仍然更偏“可长期维护、行为可预测”，而不是靠一堆隐藏 fallback 堆出表面上的全能

所以这篇不算“宣告已经做完”，更像一次阶段更新。

## 如果你想试一下

最直接的方式还是：

```bash
npm i -g nextclaw
nextclaw start
```

打开 `http://127.0.0.1:55667` 即可。

如果你更偏桌面安装路线，现在也已经比 3 月底那会儿更完整了。
