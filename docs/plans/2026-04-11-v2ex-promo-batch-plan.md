# 2026-04-11 V2EX 宣传批次规划

## 背景

- 上一轮真实对外宣传基线不是 `2026-03-06`，而是 `2026-03-24` 的 V2EX 功能更新帖。
- 历史补录文件为 `docs/marketing/2026-03-24-v2ex-nextclaw-feature-update-post.md`。
- `2026-03-24` 那篇已经对外讲过的点包括：
  - 微信 clawbot
  - 远程访问
  - 授权分享
  - codex
  - claude code
  - 开启认证
- 从 `2026-03-24 23:59` 到 `2026-04-11`，仓库大约新增了 `357` 个 commit。
- 所以下一篇不该继续把上面这些能力当“新增功能”重复广播，而应讲“这之后哪些东西继续产品化、继续补齐、继续往前走了”。

## 这轮宣传目标

1. 让看过 `2026-03-24` 那篇帖子的人感到项目确实继续往前走了，而不是把同一批功能换个标题再发一次。
2. 把 `357` 个 commit 压缩成少数几个用户可感知的进展，而不是堆 changelog。
3. 把 NextClaw 从“功能更新帖”推进到“产品在往个人 AI 操作层继续成形”的认知。
4. 收集下一轮优先级反馈，重点看大家更在意远程访问、多 Agent、桌面体验，还是渠道与自动化。

## 受众判断

- 第一层：关注本地 AI、自动化、Agent、开源工具的 V2EX 用户。
- 第二层：对 OpenClaw、Claude Code、Codex、自动化工作流感兴趣，但不想自己拼太多碎片的人。
- 第三层：已经看过第一次帖子，想判断项目是否真的持续推进的人。

## 可选叙事角度

### 方案 A：3 月 24 日之后的阶段进展复盘

- 讲法：上次发帖后，这两周多重点补了哪些关键台阶。
- 优点：最自然，可信度高，容易承接 `357 commits` 这个事实。
- 风险：如果写成流水账，会显得像 changelog。

### 方案 B：从替代品到个人 AI 操作层

- 讲法：不再只讲“兼容 OpenClaw + 更好上手”，而是强调统一入口、远程访问、多 Agent、跨设备继续。
- 优点：更接近长期愿景，认知升级更明显。
- 风险：如果缺少具体证据，容易显得太虚。

### 方案 C：先讲一个用户价值最高的能力

- 讲法：主打“本地跑起来的 NextClaw，现在可以从网页继续打开”，或者主打“多 Agent 开始真正成形”。
- 优点：记忆点强。
- 风险：会把其它重要进展压没，不适合这轮“阶段复盘”。

## 推荐方案

- 采用 `方案 A + 方案 B` 的混合打法。
- 正文主线用“3 月 24 日之后的阶段进展复盘”承载可信度。
- 叙事落点用“NextClaw 正在越来越像个人 AI 操作层”拉高认知。
- 不推荐纯“功能清单帖”，也不推荐纯“愿景帖”。

## 传播学执行要求

- 标题和开头必须保留高势能锚点：
  - `Codex`
  - `Claude Code`
  - `远程访问`
  - 如空间允许，再带上 `微信`
- 正文不能把 `Codex / Claude Code` 写成“已支持，略”，而要写成：
  - 上一轮已经接进来
  - 这一轮继续补深
  - 现在更接近真实可用
- 结构顺序默认应为：
  1. 高势能锚点
  2. 不是旧功能重发，而是继续补深
  3. 具体证据
  4. 产品方向升级
  5. CTA 与反馈问题
- 不要一上来先讲抽象愿景。
- 不要先讲“桌面、架构、维护性治理”这种低传播势能信息。
- 愿景要作为升维解释，放在中后段，而不是首屏。

## 这次 V2EX 帖子的核心信息

1. 上次发帖是 `2026-03-24`，那时已经讲过微信 clawbot、远程访问、codex、claude code、认证。
2. 这次最值得讲的，不是“又加了几个功能名词”，而是这些能力之后又继续往产品化推进了：
   - 远程访问从“能用”继续推进到“更像正式产品”
   - 多 Agent / child session 开始具备更明确的产品语义
   - codex / claude code 不再只是“能接进来”，而是继续补兼容性、路由和真实使用链路
   - 桌面分发与安装路线明显更完整
   - 聊天、附件、项目上下文等日常体验继续收敛
   - 渠道、Marketplace、自动化底层继续往统一 runtime 收敛
3. 项目定位也比上次更清晰：
   - 不是只做聊天壳
   - 不是功能垃圾场
   - 而是想做 AI 时代的个人操作层

## 证据优先级

- `Remote Access` 已从“有这个能力”继续推进到 UI 主路径、账号入口和 `NextClaw Web` 打开流程。
- `nextclaw@0.17.0` 已作为多 Agent 对外语义升级版本发布。
- `codex` 继续补了图片输入、模型路由和本地插件开发验证入口。
- `claude code` 继续补了 marketplace 安装、provider 路由和 OpenAI-compatible bridge。
- Linux desktop 已补 `.deb` 与 APT 仓库。
- 桌面 launcher / bundle 更新基础层已经开始落地。
- 图片、附件、项目上下文、文件预览 / diff 等日常使用链路继续补齐。

### 适合在回帖或跟进帖里补充的证据

- 具体包版本变化。
- 细碎 UI polish。
- 维护性治理、NCP 收敛、内部架构清理。

## 不建议这轮过度展开的内容

- 不要再把微信 clawbot、远程访问、codex、claude code、认证本身写成“这次新增功能”，因为上一轮已经讲过。
- 不要把大量内部治理规则当成对外卖点。
- 不要把每个 release batch 都写进去。
- 不要过度承诺“已经全都完成”，尤其是桌面更新与远程访问后续还会继续打磨。
