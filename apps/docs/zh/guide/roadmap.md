# NextClaw 路线图

本文档描述 NextClaw 当前阶段的具体优先级与执行计划。所有工作服务于[产品愿景](./vision.md)中的四大核心支柱。迭代历史见：[docs/logs](https://github.com/Peiiii/nextclaw/blob/master/docs/logs/README.md)。

如果你想先看一个更直观的公开页面，了解最近发版节奏、代码规模变化、产品笔记和当前界面截图，先看 [Project Pulse](./project-pulse.md)。

---

## 当前阶段优先级

### 1. 自知能力（Agent 认识自己）

> 服务于愿景：自知与自治

- Agent 能查询自身版本、配置、运行状态、健康度、能力边界与文档入口。
- 涵盖静态信息（配置、版本）与动态信息（运行时状态、会话上下文）。
- 在对话中回答"我有哪些渠道？"、"当前模型是什么？"、"我健康吗？"等问题。

### 2. 自治能力（Agent 管理自己）

> 服务于愿景：自知与自治

- Agent 可通过工具执行管理动作：改配置、启停渠道、增删 cron、安装/卸载插件、诊断/重启等。
- 高风险操作需确认机制。
- 用户通过自然语言完成原本需要 UI/CLI 的管理操作。

### 3. 插件 SDK 与开发流程

> 服务于愿景：插件化拓展与生态繁荣

- 稳定插件 SDK 与 OpenClaw 兼容的开发流程。
- 明确内核与插件的边界；核心 API 趋于收敛。
- 支持从 UI 中安装、启用、配置、卸载插件。

### 4. 开箱即用体验

> 服务于愿景：开箱即用

- 首次启动展示核心能力与典型场景。
- 预置场景模板，支持一键启用。
- 优化从安装到首个可用场景的路径。

## 下一阶段方向

### Marketplace 生态

> 服务于愿景：插件化拓展与生态繁荣

- 用户插件/Skill 发布与发现流程。
- 评价、版本管理、依赖解析等基础设施。
- 实现"使用 → 开发 → 分享 → 发现"闭环。

### 宿主系统与数据能力

> 服务于愿景：数字世界全能基础设施

- 工具体系拓展：文件管理、进程管理、系统监控等宿主系统操作。
- 互联网数据接入：搜索、抓取、API 调用、信息聚合。
- 用户数据连接：整合散布在各 App/平台中的数据。

### 多 Agent 与多实例

- 多 Agent：会话隔离、bindings、路由与运行时语义增强。
- 多实例：部署模型、配置边界与最佳实践沉淀。

## 持续关注

- **Landing 页面**：产品价值表达与关键入口优化。
- **UI 与交互**：配置 UI、信息架构、统一交互体验。
- **OpenClaw 兼容性**：跟进插件 SDK 与渠道协议。
- **Cron / Heartbeat**：强化自动化任务与文档。
- **可观测性与运维**：`doctor`、`status`、配置热重载、部署文档。
- **文档与教程**：安装、配置、首个 bot、首个 cron 等完整路径。
