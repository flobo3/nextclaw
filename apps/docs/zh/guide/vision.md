# NextClaw 产品愿景

NextClaw 的长期愿景，是成为 AI 时代的个人操作层。

它不是传统意义上的操作系统内核，而是用户使用软件、互联网、系统、服务与云计算的默认入口。

这是一份面向文档站的公开版愿景说明，默认与仓库内的上位文档 `docs/VISION.md` 保持对齐。

## 它到底想做什么

NextClaw 想做的，不是一个“功能很多的 AI 产品”，而是一个足够统一、足够自然、足够强大、足够可长期依赖的 AI 原生入口。

理想状态下，用户不需要先想“我要打开哪个软件、哪个网站、哪个服务”，而是先进入 NextClaw，说出目标，由 NextClaw 去理解意图、调用工具、连接服务、组织流程，并把结果交付出来。

如果传统操作系统定义了“人如何使用计算机”，那么 NextClaw 想定义的是“人如何在 AI 时代使用数字世界”。

## 这份愿景该怎么理解

“个人操作层”是上位目标，但它不是空话。要把它真正做成产品，NextClaw 需要同时满足下面几条主线。

### 1. 统一入口

NextClaw 首先要成为统一入口，而不是又一个平行工具。

- 用户进入数字世界时，应该越来越倾向于先打开 NextClaw
- 软件、服务、渠道、设备与云资源，应该尽量通过 NextClaw 被统一接入与管理
- 它应该逐步成为能承接工作流、沟通流和信息流的默认工作台

### 2. 能力编排

NextClaw 的价值不在于把所有功能都自己重做一遍，而在于更好地连接、调度、组合现有的软件、系统、服务与算力。

- 把用户意图翻译成可执行流程
- 调用工具、连接服务、组织多步任务
- 在多模型、多渠道、多插件、多运行环境之间收敛成统一体验

我们追求的是“统一入口与能力编排”，不是“功能垃圾场”。

### 3. 自感知与自治

原本的“自知自治”方向不应该丢掉，但术语上没必要拆成两个近义词。现在统一收口成“自感知与自治”。

- 知道自己是谁：身份、版本、规则、能力边界、工具边界是什么
- 知道自己当前的系统状态：完整配置、运行状态、健康度、服务连接状态是什么
- 知道自己在为谁工作：用户是谁、偏好是什么、历史习惯是什么
- 知道自己处在什么场景：当前会话、项目、渠道、设备、运行环境是什么
- 知道自己做过什么、结果如何、哪里做错了
- 能通过工具逐步执行系统内关键管理动作，例如配置修改、渠道启停、任务管理、扩展管理、诊断恢复

目标不是“神化 agent”，而是让用户通过自然语言与统一控制面，就能理解和掌控 NextClaw 自身。

### 4. 自进化

NextClaw 也不能是一个静态入口。它应该具备可治理、可积累、可复用的自进化能力，也就是：

- 能从错误中学习，而不是反复犯同样的问题
- 能从用户反馈、任务结果、失败回放里做反思和修正
- 能逐步沉淀更好的执行策略、工具使用纪律、上下文组织方式和记忆结构
- 能把一次次经验沉淀为更长期的用户偏好、项目知识、procedural memory 与 skill memory
- 能随着使用时间增长，越来越像“这个用户自己的操作层”，而不是永远停留在通用默认形态

### 5. 插件化拓展与生态繁荣

个人操作层不能靠主产品自己无限膨胀，而要靠稳定内核加扩展生态来成长。

- 核心运行时、插件 SDK、渠道协议需要逐步收敛并保持稳定边界
- 长期扩展能力应更多通过 plugin、skill、marketplace 来完成
- 用户与开发者应该能形成“使用 -> 开发 -> 分享 -> 发现”的闭环

### 6. 数字世界基础设施

NextClaw 不只是一个聊天机器人，它致力于成为用户在数字世界中的通用操作基础设施：

- 理解和利用宿主系统、本地服务、文件系统与硬件资源
- 接入互联网数据源、API 与云计算资源
- 统一管理用户散落在各类系统中的数据

### 7. 开箱即用

“个人操作层”不能只对高阶用户成立，也必须对新用户成立。

- 首次启动就应让用户知道它能做什么
- 关键场景应支持一键配置与启动
- 从了解、安装到第一次真正跑通，应尽可能短、尽可能顺滑

## 我们追求的产品级别

我们追求的不是复刻电影里的 JARVIS 设定，而是接近那种产品级别的存在：

- 足够重要，能成为用户的第一入口
- 足够统一，能承接软件、服务、平台与算力
- 足够自然，让用户更多表达意图，少做工具切换
- 足够连续，不会每一轮都像失忆
- 足够可靠，让人愿意长期把它当成默认工作台
- 足够会成长，越用越贴近用户而不是长期停在原地

## 这不意味着什么

这份愿景不是说：

- 要把所有能力都硬塞进一个单体产品
- 要把一切都自己重做一遍
- 要为了看起来全能而持续堆功能
- 要做一个只有聊天外壳、却不能真正调度世界的 AI
- 要把“自感知”写成玄学叙事，却没有真实系统能力支撑
- 要把“自进化”做成不可控的自动自我修改黑盒

## 与 NCP 的关系

`NextClaw` 是面向用户的产品化操作层，`NCP` 是支撑这个操作层的通用协议、运行时与积木体系。

可以把 NCP 理解为基础设施底座，把 NextClaw 理解为面向真实用户的产品化入口。
