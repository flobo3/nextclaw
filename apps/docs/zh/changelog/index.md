# 发布日志

`发布日志` 用来记录已经真正发出去的版本。
它比 `Notes` 更技术、更完整，但又不等于把原始包元数据直接堆给用户看。

如果你想知道“到底发了什么、哪些包一起动了、怎么验证的”，应该先看这里。

## 最新发布

## 定位说明

- `Notes`：短更新，强调用户感知与价值。
- `发布日志`：记录真实发版范围、版本号和验证方式。
- `博客`：解释更长线的判断、设计取舍与产品思考。
